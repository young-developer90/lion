راهنمای ابزارهای GUI

برنامه‌های دسکتاپ بومی با ابزارهای GUI چندپلتفرمی زمین بسازید.

نمای کلی

زمین با دو ابزار GUI ارائه می‌شود که از API یکسانی استفاده می‌کنند، بنابراین کد نوشته شده برای یکی بدون تغییر روی دیگری کار می‌کند:

  • Sol (ویندوز) — یک ابزار مبتنی بر Win32 که به صورت ماژول داخلی در ویندوز ارائه می‌شود. هیچ وابستگی خارجی نیاز ندارد.
  • Luna (لینوکس) — یک ابزار مبتنی بر GTK4 که در پشت پرچم ویژگی luna در دسترس است. نیاز به کتابخانه‌های توسعه GTK4 در سیستم میزبان دارد.

هر دو ابزار نام‌های تابع و سازنده‌های ویجت یکسانی را ارائه می‌دهند. شما ابزار متناسب با پلتفرم خود را انتخاب می‌کنید و بقیه کد شما یکسان می‌ماند.

💡 توجه

در ویندوز، sol() همیشه در دسترس است. در لینوکس، از luna() استفاده کنید و با --features luna بسازید.

شروع سریع

در اینجا یک برنامه GUI حداقلی که یک پنجره با یک برچسب نمایش می‌دهد:

let gui = sol()  // or luna() on Linux
let win = gui.Leo("My App", 400, 300)
let label = gui.Label(win, "Hello, Zamin!")
gui.pack(label)
gui.mainloop(win)

این را به عنوان app.zamin ذخیره کنید و اجرا کنید:

$ zamin run app.zamin

یک پنجره با عنوان "My App" با متن "Hello, Zamin!" در وسط ظاهر می‌شود.

مرجع ویجت‌ها

Leo (پنجره)

پنجره سطح بالا که همه ویجت‌های دیگر را در خود نگه می‌دارد.

امضاتوضیحات
Leo(title?, width?, height?) یک پنجره سطح بالا جدید ایجاد می‌کند. اگر عرض/ارتفاع حذف شوند، پیش‌فرض 640×480 است. 
let gui = sol()
let win = gui.Leo("Settings", 800, 600)

Frame

یک ویجت ظرف برای گروه‌بندی و سازماندهی ویجت‌های فرزند.

امضاتوضیحات
Frame(parent) یک فریم در داخل ویجت والد داده شده ایجاد می‌کند. 
let gui = sol()
let win = gui.Leo("App", 400, 300)
let frame = gui.Frame(win)
gui.pack(frame)

Label

متن ایستا نمایش می‌دهد. ورودی کاربر را نمی‌پذیرد.

امضاتوضیحات
Label(parent, text?) یک برچسب با متن داده شده در داخل ویجت والد ایجاد می‌کند. 
let gui = sol()
let win = gui.Leo("App", 400, 300)
let lbl = gui.Label(win, "Status: Ready")
gui.pack(lbl)

Button

یک دکمه قابل کلیک که با فشار دادن یک callback را فعال می‌کند.

امضاتوضیحات
Button(parent, text?, callback?) یک دکمه با برچسب و handler کلی اختیاری ایجاد می‌کند. 
let gui = sol()
let win = gui.Leo("App", 400, 300)

func on_click() {
    print("Button was clicked!");
}

let btn = gui.Button(win, "Click Me", on_click)
gui.pack(btn)
gui.mainloop(win)

Callbackها توابع معمولی زمین هستند. می‌توانند آرگومان بپذیرند یا closureهایی باشند که وضعیت محلی را ضبط می‌کنند.

Entry

یک فیلد ورود متن تک‌خطی برای ورودی کاربر.

امضاتوضیحات
Entry(parent) یک فیلد ورود متن در داخل ویجت والد ایجاد می‌کند. 
let gui = sol()
let win = gui.Leo("App", 400, 300)
let entry = gui.Entry(win)
gui.pack(entry)

مدیریت چیدمان

ابزارهای GUI زمین دو روش چیدمان برای مرتب‌سازی ویجت‌ها ارائه می‌دهند:

pack

امضاتوضیحات
pack(widget, side?, padx?, pady?) یک ویجت را با استفاده از مدیر هندسی pack قرار می‌دهد. side می‌تواند "top"، "bottom"، "left"، یا "right" باشد. padx و pady به ترتیب فاصله افقی و عمودی را بر حسب پیکسل اضافه می‌کنند. 
gui.pack(label, "top", 10, 5)
gui.pack(btn, "left", 5, 5)

place

امضاتوضیحات
place(widget, x, y, w?, h?) یک ویجت را در مختصات مطلق قرار می‌دهد. w و h عرض و ارتفاع را در صورت تمایل تنظیم می‌کنند. 
gui.place(label, 20, 50, 200, 30)

عملیات ویجت‌ها

این توابع ویجت‌ها را پس از ایجاد تغییر می‌دهند یا از آنها پرس‌وجو می‌کنند.

تابعتوضیحات
config(widget, property, value) تنظیم یک ویژگی روی ویجت. ویژگی‌های رایج: "text"، "command"، "font"، "fg"، "bg".
get(entry) محتوای متنی جاری یک ویجت Entry را برمی‌گرداند.
insert(entry, pos, text) متن را در موقعیت داده شده در یک ویجت Entry درج می‌کند.
delete(entry, start, end?) کاراکترها را از start تا end در یک ویجت Entry حذف می‌کند. اگر end حذف شود، یک کاراکتر حذف می‌کند.
title(window, text) عنوان یک پنجره را تغییر می‌دهد.
geometry(window, w, h) اندازه یک پنجره را به عرض و ارتفاع داده شده تغییر می‌دهد.
destroy(widget) یک ویجت را از چیدمان حذف کرده و منابع آن را آزاد می‌کند.
messagebox(text, title?) یک دیالوگ پیام modal با متن داده شده و عنوان اختیاری نمایش می‌دهد. 
gui.config(btn, "text", "Updated Label")
let value = gui.get(entry)
gui.title(win, "New Title")
gui.messagebox("Operation complete!", "Info")

مثال: ماشین حساب

یک برنامه ماشین حساب کامل با فیلد نمایش و دکمه‌های حسابی:

let gui = sol()
let win = gui.Leo("Calculator", 280, 350)

let display = gui.Entry(win)
gui.pack(display, "top", 10, 10)

let expression = ""

func on_digit(d) {
    expression = expression + d;
    gui.config(display, "text", expression);
}

func on_operator(op) {
    expression = expression + " " + op + " ";
    gui.config(display, "text", expression);
}

func on_clear() {
    expression = "";
    gui.config(display, "text", "");
}

func on_equal() {
    let result = eval(expression);
    gui.config(display, "text", str(result));
    expression = str(result);
}

// Build the button row for digits 1-9
for row in 0..3 {
    let frame = gui.Frame(win);
    gui.pack(frame, "top");
    for col in 0..3 {
        let digit = str(row * 3 + col + 1);
        let d = digit;
        let btn = gui.Button(frame, digit, func() { on_digit(d) });
        gui.pack(btn, "left", 2, 2);
    }
}

// Bottom row: 0, clear, equals
let bottom = gui.Frame(win)
gui.pack(bottom, "top")
let b0 = gui.Button(bottom, "0", func() { on_digit("0") })
gui.pack(b0, "left", 2, 2)
let bc = gui.Button(bottom, "C", on_clear)
gui.pack(bc, "left", 2, 2)
let be = gui.Button(bottom, "=", on_equal)
gui.pack(be, "left", 2, 2)

// Operator buttons
let ops = gui.Frame(win)
gui.pack(ops, "top")
for op in ["+", "-", "*", "/"] {
    let o = op;
    let b = gui.Button(ops, op, func() { on_operator(o) });
    gui.pack(b, "left", 2, 2);
}

gui.mainloop(win)

مثال: ویرایشگر متن

یک ویرایشگر متن پایه با فیلد ورودی، دکمه باز کردن و دکمه ذخیره:

let gui = sol()
let win = gui.Leo("Text Editor", 600, 400)

// File path entry
let top_frame = gui.Frame(win)
gui.pack(top_frame, "top", 5, 5)

gui.Label(top_frame, "File:")
let path_entry = gui.Entry(top_frame)
gui.pack(path_entry, "left", 5, 5)

// Text area (simulated with a large Entry)
let text_area = gui.Entry(win)
gui.pack(text_area, "top", 10, 10)

// Button bar
let btn_frame = gui.Frame(win)
gui.pack(btn_frame, "top", 5, 5)

func on_open() {
    let path = gui.get(path_entry);
    if path == "" {
        gui.messagebox("Enter a file path first.", "Error");
        return;
    }
    let content = read(path);
    gui.config(text_area, "text", content);
    gui.title(win, "Text Editor - {path}");
}

func on_save() {
    let path = gui.get(path_entry);
    if path == "" {
        gui.messagebox("Enter a file path first.", "Error");
        return;
    }
    let content = gui.get(text_area);
    write(path, content);
    gui.messagebox("File saved!", "Success");
}

let open_btn = gui.Button(btn_frame, "Open", on_open)
gui.pack(open_btn, "left", 5, 5)

let save_btn = gui.Button(btn_frame, "Save", on_save)
gui.pack(save_btn, "left", 5, 5)

gui.mainloop(win)

یادداشت‌های پلتفرم

Sol (ویندوز)

  • مستقیماً بر روی Win32 API ساخته شده است — بدون وابستگی‌های خارجی یا کتابخانه‌های زمان اجرا.
  • به عنوان یک ماژول داخلی در ساخت‌های ویندوزی زمین ارائه می‌شود.
  • از ظاهر و حس استاندارد ویندوز با رندر بومی ویجت پشتیبانی می‌کند.

Luna (لینوکس)

  • بر اساس GTK4 برای محیط‌های دسکتاپ مدرن لینوکس ساخته شده است.
  • نیازمند نصب کتابخانه‌های توسعه GTK4 است:
# Debian / Ubuntu
sudo apt install libgtk-4-dev

# Fedora
sudo dnf install gtk4-devel

# Arch
sudo pacman -S gtk4
  • زمین را با ویژگی luna فعال بسازید:
cargo build --release --features luna

macOS

  • پشتیبانی از macOS برای نسخه آینده برنامه‌ریزی شده است. امروز در macOS می‌توانید از ماژول‌های کتابخانه استاندارد برای ساخت ابزارهای خط فرمان استفاده کنید.
✔ نکته

از آنجایی که Sol و Luna از API یکسانی استفاده می‌کنند، می‌توانید کد GUI خود را یک بار بنویسید و با انتخاب سازنده ابزار مناسب در بالای فایل، آن را برای پلتفرم‌های مختلف کامپایل کنید.
