Advanced Guide

This guide covers advanced features of the Lion language including extending Lion with C, interopping with Python, leveraging GPU acceleration via CUDA, embedding Lion in Rust applications, and using the built-in LSP server.

C Extensions

Lion supports native C extensions for performance-critical code or wrapping existing C libraries. Extensions are compiled as shared objects and loaded automatically at runtime.

Header File

Include the main header in your C source:

#include "lion.h"

The header is located at include/lion.h in the Lion source tree.

Lion Types

Lion exposes the following C type constants for use with the LionValue union:

ConstantDescription
LION_NILNil / null value
LION_INTInteger (64-bit signed)
LION_FLOATDouble-precision float
LION_BOOLBoolean true/false
LION_STRINGHeap-allocated string

LionValue Struct

The LionValue struct is a tagged union that represents all values that can cross the C/Lion boundary:

typedef struct {
    int type;
    union {
        int64_t i;
        double f;
        int b;
        const char *s;
    } as;
} LionValue;

LionModuleFunc Struct

Each exported function is described by a LionModuleFunc entry:

typedef struct {
    const char *name;
    LionValue (*func)(int argc, LionValue *argv);
    int arity;  // -1 for variadic
} LionModuleFunc;

Exporting a Module

Every C extension must export a lion_module_init function that returns a LionModuleFunc array and sets the count via the provided pointer:

 LionModuleFunc *lion_module_init(int *count) {
    static LionModuleFunc funcs[] = { ... };
    *count = sizeof(funcs) / sizeof(funcs[0]);
    return funcs;
}

Building

Compile your extension as a shared library:

gcc -shared -fPIC -O2 -I../include -o mymod.so mymod.c

Place the resulting .so file in your project's modules/ directory. Lion will auto-load it when you write use mymod.

Full Example: math_ext

#include "lion.h"

static LionValue math_add(int argc, LionValue *argv) {
    if (argc != 2) {
        return (LionValue){ LION_NIL, { .i = 0 } };
    }
    int64_t a = argv[0].as.i;
    int64_t b = argv[1].as.i;
    return (LionValue){ LION_INT, { .i = a + b } };
}

static LionValue math_multiply(int argc, LionValue *argv) {
    if (argc != 2) {
        return (LionValue){ LION_NIL, { .i = 0 } };
    }
    int64_t a = argv[0].as.i;
    int64_t b = argv[1].as.i;
    return (LionValue){ LION_INT, { .i = a * b } };
}

LionModuleFunc *lion_module_init(int *count) {
    static LionModuleFunc funcs[] = {
        { "add",      math_add,      2 },
        { "multiply",  math_multiply,  2 },
    };
    *count = sizeof(funcs) / sizeof(funcs[0]);
    return funcs;
}

Usage in Lion:

use math_ext

let result = math_ext.add(10, 20)
print(result)  // 30

let product = math_ext.multiply(6, 7)
print(product)  // 42

Python Interop

Lion can call Python modules directly when compiled with the python feature. This enables reusing the vast Python ecosystem from Lion code.

Building with Python Support

cargo build --features python

Importing Python Modules

Use py.import() to bring a Python module into scope:

let np = py.import("numpy")

let arr = np.array([1, 2, 3, 4, 5])
print(arr.mean())  // 3.0

Checking the Python Version

print(py.version())  // e.g. "3.14.0"

Type Marshalling

Lion automatically converts types between the two languages:

Lion TypePython Type
intint
floatfloat
stringstr
boolbool
listlist
nilNone
dictdict

Examples

Using requests to fetch a URL:

let requests = py.import("requests")

let resp = requests.get("https://httpbin.org/json")
print(resp.status_code)
print(resp.json())

Using numpy for matrix operations:

let np = py.import("numpy")

let a = np.array([[1, 2], [3, 4]])
let b = np.array([[5, 6], [7, 8]])
let c = np.dot(a, b)
print(c)
// [[19 22]
//  [43 50]]

CUDA / GPU Acceleration

Lion can leverage NVIDIA GPUs for computation when CUDA is available. The build system automatically detects CUDA during compilation via build.rs.

Automatic Detection

During cargo build, build.rs checks for the nvcc compiler and CUDA toolkit. If found, GPU modules are compiled and linked automatically. No extra flags are needed.

LINUM Module

The LINUM (Lion Numerical) module provides GPU-accelerated neural network primitives:

  • linum.tensor() - Create GPU tensors
  • linum.matmul() - Matrix multiplication on GPU
  • linum.relu() / linum.sigmoid() - Activation functions
  • linum.softmax() - Softmax for classification
  • linum.cross_entropy() - Loss functions

GPU Memory API

print(linum.gpu_name())        // "NVIDIA GeForce RTX 4090"
print(linum.gpu_memory())      // 24576 (MB)
print(linum.gpu_memory_used()) // 512 (MB)

Example: Neural Network

use linum

let model = linum.Sequential([
    linum.Linear(784, 256),
    linum.ReLU(),
    linum.Linear(256, 128),
    linum.ReLU(),
    linum.Linear(128, 10),
    linum.Softmax()
])

let optimizer = linum.Adam(model.parameters(), lr: 0.001)
let loss_fn = linum.CrossEntropyLoss()

for epoch in 0..10 {
    let pred = model.forward(train_data)
    let loss = loss_fn(pred, train_labels)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    print("Epoch {epoch}, Loss: {loss.item():.4f}")
}

Embedding in Rust

Lion can be embedded as a library in Rust applications. This is useful for building tools, IDEs, or applications that need scripting capabilities.

Cargo.toml Dependency

[dependencies]
lion = { path = "../lion" }

API

Two primary functions are provided:

  • lion::execute_source(code: &str) - Execute a Lion source string
  • lion::execute_file(path: &str) - Execute a Lion file from disk

Full Rust Embedding Example

fn main() {
    // Execute a Lion source string
    lion::execute_source(r#"
        let greeting = "Hello from Lion!"
        print(greeting)
        let result = 2 + 3
        print("2 + 3 = {result}")
    "#).expect("Failed to execute Lion source");

    // Execute a Lion file
    lion::execute_file("scripts/app.lion").expect("Failed to execute file");

    // Access Lion values from Rust
    let mut vm = lion::Vm::new();
    vm.execute("let x = 42").unwrap();
    let value = vm.get_global("x").unwrap();
    println!("Lion returned: {:?}", value);
}

LSP Server

Lion ships with a built-in Language Server Protocol (LSP) implementation that provides IDE features for any editor with LSP support.

The lion-lsp Binary

After building Lion, the lion-lsp binary will be available in the output directory. It communicates over stdio using the LSP protocol.

Features

  • Completion - Context-aware autocompletion for keywords, built-in functions, module APIs, and user-defined variables
  • Hover - Type information and documentation on hover for all symbols
  • Diagnostics - Real-time syntax errors, type mismatches, and undefined references

VS Code Extension

A VS Code extension is available. Install it from the marketplace and add the following to your settings.json:

{
  "lion.server.path": "/path/to/lion-lsp"
}

The extension provides syntax highlighting, error squiggles, autocompletion popups, and hover documentation out of the box.
